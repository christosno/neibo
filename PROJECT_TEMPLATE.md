# Project Template Setup Guide

This document helps you create a new project with the same setup as Plantly.

## Quick Setup Script

Run the included script:
```bash
./create-new-project.sh <project-name> <project-path>
```

Example:
```bash
./create-new-project.sh myapp ../myapp
```

## Manual Setup

If you prefer to set up manually, follow these steps:

### 1. Create New Expo Project
```bash
npx create-expo-app@latest myapp --template blank-typescript
cd myapp
```

### 2. Copy Configuration Files

Copy these files from this project to your new project:

#### Essential Config Files:
- `tsconfig.json` - TypeScript configuration with path aliases
- `eslint.config.js` - ESLint configuration
- `.gitignore` - Git ignore rules
- `package.json` - Dependencies and scripts (update name)
- `app.json` - Expo configuration (update name, slug, bundle IDs)
- `index.ts` - Entry point

### 3. Install Dependencies

```bash
npm install
```

### 4. Update Configuration

#### Update `package.json`:
- Change `name` to your project name

#### Update `app.json`:
- `name`: Display name
- `slug`: URL-friendly name
- `scheme`: Deep linking scheme
- `ios.bundleIdentifier`: iOS bundle ID (e.g., `com.yourname.appname`)
- `android.package`: Android package name (e.g., `com.yourname.appname`)

### 5. Project Structure

The basic structure includes:
```
app/
  _layout.tsx          # Root layout
  (tabs)/              # Tab navigation
components/            # Reusable components
store/                 # Zustand stores
theme.ts              # Theme configuration
```

### 6. Key Dependencies

This project uses:
- **Expo Router** - File-based routing
- **Zustand** - State management
- **TypeScript** - Type safety
- **ESLint + Prettier** - Code quality
- **Expo Dev Client** - Custom development client

### 7. Initialize Git

```bash
git init
git add .
git commit -m "Initial commit"
```

## Configuration Details

### TypeScript (`tsconfig.json`)
- Extends Expo base config
- Strict mode enabled
- Path alias `@/*` for root imports

### ESLint (`eslint.config.js`)
- Uses Expo ESLint config
- Prettier integration
- Ignores `dist/*`

### Expo (`app.json`)
- New Architecture enabled
- Expo Router plugin
- Custom fonts support
- iOS and Android configurations

## Notes

- The `ios/` and `android/` folders are generated by Expo and should be in `.gitignore`
- Run `npx expo prebuild` if you need native folders
- Assets (icons, splash screens) should be updated for your project
